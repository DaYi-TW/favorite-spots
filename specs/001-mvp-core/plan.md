# Implementation Plan: MVP Core

**Branch**: `001-mvp-core` | **Date**: 2026-04-18 | **Spec**: [spec.md](spec.md)

## Summary

Build the MVP of Favorite Spots Knowledge Graph: user authentication, spot CRUD with AI-powered link parsing, Redis parse caching, and a mobile-first card feed with filters. Backend in Spring Boot 3 / Java 21 with PostgreSQL; frontend in Next.js 14 (App Router) with Tailwind CSS following the Vivid Atlas design system.

## Technical Context

**Language/Version**: Java 21 (backend), TypeScript / Node 20 (frontend)
**Primary Dependencies**: Spring Boot 3.x, Spring Security, Spring Data JPA, Flyway, Jsoup, Anthropic Java SDK (Claude API); Next.js 14, Tailwind CSS, Axios
**Storage**: PostgreSQL 16 (primary data), Redis 7 (parse cache, TTL 24h)
**Testing**: JUnit 5 + MockMvc (backend), Jest + React Testing Library (frontend)
**Target Platform**: Docker Compose local dev; mobile browser (iOS Safari 15+, Android Chrome 105+)
**Project Type**: Web application (Spring Boot REST API + Next.js frontend)
**Performance Goals**: Parse preview ≤10s first-call, ≤1s cached; feed load ≤2s for 200 spots; filter/search ≤500ms
**Constraints**: Claude API calls cached per normalized URL; cover images stored as external URLs (no upload/proxy in Phase 1); Neo4j deferred to Phase 2
**Scale/Scope**: Single-user MVP, ~200 spots per user

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Gate | Status |
|-----------|------|--------|
| I. Zero-Friction Data Entry | URL input is primary path; manual form is fallback only; Redis cache prevents re-parsing | ✅ Pass |
| II. Dual-Store Architecture | Neo4j sync deferred to Phase 2 per Principle V — justified | ✅ Pass |
| III. AI-First Parsing | Jsoup meta → Claude API → manual form, strictly in that order | ✅ Pass |
| IV. Mobile-First Intelligent Curator | Tailwind config uses Vivid Atlas tokens from mockup HTML; all screens match reference mockups | ✅ Pass |
| V. Phased Delivery | Phase 1 only — no graph, social, map, or extension features | ✅ Pass |

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|--------------------------------------|
| Two external calls per parse (Jsoup + Claude API) | Jsoup extracts structured meta efficiently; Claude handles unstructured text and multi-language content | Jsoup alone fails on pages without OG tags; Claude alone is too expensive without the cheap Jsoup pre-pass |
